Design Process
  • Problem Statement and Research
    • Clearly define the problem and conduct user research to understand the users needs, motivations, and behaviours
    • Study competitors and trends in the current market
    • Create user personas representing the primary user groups
  • Project Goals and Planning
    • Map out user journeys and use cases to understand how the user interacts with the product
    • Define key design goals based on the problem statement, user needs, and business goals
    • Plan the structure of the information to ensure its logical and intuitive
  • Mockups and Design
    • High level sketches and wireframes to outline basic design elements
    • Develop low level prototypes that include all interactive components, this should be close to the final design and consider design elements such as typography, colours, and imagery
    • Ensure considerations such as accessibility are taken into account in the final design
  • User Testing and Feedback
    • Craft a survey that considers ..... potential downfalls
    • Gather feedback from users on usibility and overall experience
    • Based on the feedback from the user testing phase make necessary changes
  • Documentation and Design Handoff
    • Provide detailed design documents including layouts, graphics, and interactive elements
    • Colloborate closely with developers and particpate in final testing to ensure quality ensurence
  • Next Steps
    • Deploy and monitor the product monitoring user interactions
    • Gather post launch feedback and implement that into continuous iterative improvment to ensure the product continues to meet evolving user needs
Fight N' Flight
Overview
Fight N’ Flight is a two-player game developed in C++ inspired by retro arcade games like Galaga, featuring pixel-art design. Gameplay consists of constant shooting from both players. The two players must move up and down the screen to avoid the others fire. There is also a missile special attack that has a cooldown after use and other obstacles that can block the other player . The first player that’s health drops to zero losses.
Role
This project was completed by a team of six programmers using C++ and the game development library SFML. My role was design lead and UI programmer. Responsibilities of this role included designing the user interfaces and making the sure the game provides a good user experience. In addition, I designed all game assets and backgrounds as well as assisted coded the graphical user interfaces.
Tools
The group used planning and version controls softwares such as Confluence, Jira, and Github to create an Agile development work style for the team. This consisted of frequent team meetings and weekly scrums. team members developed the code using IDE's such as CLion and VSCode
Goals
The goal of this project was to create a fun and engaging game that could be played by two players. The game was designed to be simple and easy to learn but also challenging and competitive. The GUI should be easy to navigate and have clear navigational paths
Approach
Performance Indicators
The key performance indicator in terms of user experience is the user interface responsiveness. The game should launch quickly and display the menu to the user within a certain benchmark of time. All the buttons should be functional and followed by quick page load times. The most important aspect is the responsiveness of the game characters when moving and firing during gameplay as well as the health bar updating when damage is taken.
Take Aways
An important takeaway from this project was learning how to develop a project in a larger team. When not directed properly larger teams can have lower productivity and struggle to make a quality product, so this project taught me invaluable skills in teamworking and project management. This included breaking problems down into their fundamental challenges and assigning each task to the team member best suited to the task whether it be design, development or leadership roles. Learning how to use project management software’s like confluence and working in an agile development environment highlighted the importance of frequent communication and continuously incorporating feedback from other team members and testing.
Melody Mixer
The process of song creation is highly intricate, often requiring a range of expertise and coordination among musicians and technicians, which can be exclusive and inefficient. The solution aims to address this be developing neural network-based AI model capable of processing inputs from a guitar and generating a cohesive musical composition. By implementing this system with a user-friendly interface, it simplifies the music creation process but also makes it more accessible, enabling individuals without specialized skills to craft songs with the assistance of advanced AI tools.
Role
This project is a full year software development capstone project. It was completed in a group of 4 with each member having a different focus. My role on this team involved python development on the generative neural network as well as front end design and development.
Tools
Tools that were implemented for this project are the IDE VSCode, version control software Github as well as microsoft teams for collaboration. Numerous machine learning development libraries were also implemented for the technical component's implementation. The front end was developed using HTML, CSS and JQuery. The design tool Figma was used to create the wireframes and mockups for the user interface.

The goal of this project was to create a user friendly interface that allows users to create music using a guitar and the AI model.
